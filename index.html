FAQ · 05

Questions, answered

Does any data leave my server?
No. The app is a single PHP file using a local SQLite database. There are no analytics, no phone-home, and no external APIs in the data path. The only external requests are optional Google Fonts in the UI, which you can remove for fully offline use.
What do I need to run it?
PHP 8.0+ with PDO and SQLite enabled (standard on virtually all shared hosting), plus write permission in the app folder. That's the entire requirements list — no MySQL, no composer, no Node.
Can it handle large HCP lists?
Imports run in chunks (200 rows per batch by default) with an auto-reload "run until done" mode, and a cron endpoint recomputes scores and rules in batches for large datasets. SQLite comfortably handles hundreds of thousands of profiles for this workload.
How do the targeting rules work?
Rules are JSON: a condition tree (AND/OR, operators like =, >=, in, contains, regex) plus actions (set persona, add tags, set compliance flag, adjust priority score). Rules evaluate in priority order, and each profile keeps a trace of which rules fired — segments can target by that trace.
Is the scoring configurable?
Yes. Priority weights (consent / recency / engagement), the recency window, confidence penalties, imputation defaults, export field allowlists, and rate limits are all in a clearly-marked config block at the top of the file, with a few common settings overridable via .env.
What does it cost?
Nothing to deploy — it runs on infrastructure you already have. There are no per-seat licenses, no usage fees, and no contracts. The source is on GitHub.
