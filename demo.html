Data quality

Dotted values in the Profiles tab were imputed during import — gaps are filled per-field and lower the profile's confidence score, so segments can demand higher-quality data with strict mode. Custom fields from your CSV (anything beyond the standard columns) are kept per profile and usable in rules as custom.name.

Standard fields

These columns are recognized automatically (exact or fuzzy name match): hcp_id, name, email, specialty, region, organization, role, consent_email, consent_web, last_activity_ts, imports_count. Only hcp_id and name are required — and even those can be picked from any column via mapping. Everything else is imputed when missing: specialty → "General", region → "Unknown", consents → 0, with confidence penalties so nothing is silently guessed.

Custom fields

Columns that don't match a standard field are imported as custom fields — e.g. npi_number, kol_score, decile, territory. They show up on profiles, can drive rules ({"field":"custom.kol_score","op":">=","value":80}) and segment filters (custom.kol_score >= 80). The sample dataset itself ships a custom field (kol_score) to prove the path.
